# Testing Strategies for Web Applications

## Purpose
Guide developers through implementing effective testing strategies for their web applications.

## Target Audience
- Skill Level: Intermediate
- Prerequisites: JavaScript fundamentals, understanding of application architecture

## Prompt

```
I want to implement testing in my web application. Can you explain:
1. Different types of testing (unit, integration, e2e) and when to use each
2. How to set up a testing framework (Jest, Vitest, or similar)
3. Writing effective test cases and achieving good coverage
4. Mocking dependencies and external services
5. Test-driven development (TDD) principles
6. Continuous integration setup for automated testing

Please provide practical examples with modern testing tools.
```

## Example Usage
Use this prompt when setting up testing infrastructure or improving code quality and reliability.

## Expected Outcome
- Complete testing setup with working examples
- Understanding of different testing levels
- Ability to write and maintain tests
- CI/CD integration knowledge

## Tips
- Start with critical paths and edge cases
- Aim for meaningful coverage, not just high percentages
- Keep tests simple and focused
- Run tests frequently during development
- Use test coverage tools to identify gaps
