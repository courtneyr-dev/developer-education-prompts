# Git and Version Control Basics

## Purpose
Introduce beginners to Git version control and common workflows.

## Target Audience
- Skill Level: Beginner
- Prerequisites: Command line basics

## Prompt

```
I'm new to version control and need to learn Git. Please explain:
1. What is version control and why is it important?
2. Basic Git commands (init, add, commit, push, pull)
3. How to create and work with branches
4. How to resolve simple merge conflicts
5. Best practices for commit messages

Include a simple workflow example I can follow for my first project.
```

## Example Usage
Use this when starting your first coding project or joining a team that uses Git.

## Expected Outcome
- Understanding of version control concepts
- Ability to perform basic Git operations
- Confidence in managing code changes

## Tips
- Practice with a test repository first
- Commit early and often
- Write clear, descriptive commit messages
- Don't be afraid to make mistakes - Git is designed to help you recover
